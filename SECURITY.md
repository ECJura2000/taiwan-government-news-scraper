# Security Policy

## Supported Versions

Security fixes are applied to the latest version on the `main` branch.

## Reporting a Vulnerability

Do not open a public issue for vulnerabilities involving credentials, private
data, command execution, or unsafe network behavior.

Report the issue privately through GitHub's repository security advisory
feature. Include:

- affected version or commit
- reproduction steps
- expected and observed behavior
- potential impact

Do not include real credentials, private webhook URLs, or personal data in the
report. Sensitive reports will be reviewed before any public disclosure.
